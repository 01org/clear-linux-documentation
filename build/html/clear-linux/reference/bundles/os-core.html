os-core

This bundle contains the basic core components of the operating system. Clear Linux* OS for Intel® Architecture relies on systemd to provide the basic OS components. The systemd package contains solutions for:

  • Service management
  • Basic network management
  • Hostname management
  • Time synchronization management
  • Boot control management
  • Journal management

Additionally, os-core includes the util-linux and coreutils packages to provide a set of basic Linux* command line tools such as ls, cp, rm, etc.

Static IP

To configure a static IP, follow these steps:

  1. Create the file: /etc/systemd/network/50-static.network If the path does not exist, create it.

  2. Add, at the very least, the following lines to the 50-static.network file:

    [Match]
Name=<device_name>

[Network]
Address=<Provide a static IPv4 or IPv6 address with its
prefix length. Separate them with the "/" character.>
Gateway=<The gateway's address>

    The <device_name> is your network device name, for example: enp1s0 or enp0s25.

  3. To add more options you can consult the systemd network configuration manual.

  4. Restart the systemd-networkd service with the following command:

    # systemctl restart systemd-networkd

    Alternatively, restart Clear Linux.

  5. Check the new IP with the following command:

    # ip addr

Setting time

Clear Linux uses the systemd-timesyncd.service service to synchronize the system’s time. Default NTP servers are configured, for example: time1.google.com, time2.google.com, time3.google.com, and time4.google.com. Manually setting the time via timedatectl or using RTC mode is not possible. If those servers cannot be reached and the system time is incorrect, follow these steps:

  1. Set the timezone, for example, Pacific time:

    timedatectl set-timezone America/Los_Angeles

    To see a list of timezones, use the following command:

    timedatectl list-timezones

  2. Go to the /etc/systemd/ directory, if it does not exist, create it.

  3. Create the /etc/systemd/timesyncd.conf file and enter the following lines:

    [Time]
NTP=<Preferred Server>
FallbackNTP=<backup server 1> <backup server 2>

  4. Restart the timesync daemon with the following command:

    # systemctl  restart systemd-timesyncd
